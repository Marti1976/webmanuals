Příprava USB mSATA SSD

Jak nastavit RPi 3 B+ pro bootování z USB, nahrát image Home Assistant OS na SSD disk pod Windows a připravit vše pro první spuštění.

Co potřebuješ před začátkem

Checklist před startem
PC s Windows 10 nebo 11 mSATA SSD v USB pouzdře Raspberry Pi 3 B+ Internetové připojení Volitelně: malá SD karta pro USB boot aktivaci

Software který stáhneš/nainstaluješ:

  • Raspberry Pi Imager – zdarma, z raspberrypi.com/software
  • Volitelně: Raspberry Pi OS Lite pro aktivaci USB bootu (pokud ještě není povolený)

Část 1 – Povolení USB boot na RPi 3 B+

RPi 3 B+ ve výchozím nastavení bootuje ze SD karty. Aby bootoval z USB SSD, musíme toto nastavení jednou změnit – zapsat OTP (One-Time Programmable) bit do procesoru.

ℹ️ Je USB boot možná už povolen?

Novější kusy RPi 3 B+ mají USB boot povolený z výroby. Jednoduše to otestuj: zapoj USB SSD s HA image (viz část 2) bez SD karty a zkus nastartovat. Pokud RPi naběhne → přeskoč tuto část a jdi rovnou na část 2.

Jak aktivovat USB boot (pokud ještě není)

Potřebuješ malou SD kartu (stačí i 2–4 GB). Na ni nahrajeme Raspberry Pi OS Lite, bootujeme z ní jednou, systém automaticky aktivuje USB boot a pak ji vyjmeme.

1
Stáhni a nainstaluj Raspberry Pi Imager

Přejdi na raspberrypi.com/software, klikni "Download for Windows" a spusť instalátor. Průvodce: Next → Install → Finish. Spusť aplikaci.

2
Vyber zařízení a OS pro aktivaci USB bootu
  • Klikni CHOOSE DEVICE → Raspberry Pi 3
  • Klikni CHOOSE OS → Raspberry Pi OS (other) → Raspberry Pi OS Lite (32-bit)
  • Klikni CHOOSE STORAGE → vyber svoji SD kartu (NE USB SSD!)
3
Zapiš OS na SD kartu a nastartuj

Klikni NEXT → NO (bez customizace) → YES (potvrď smazání). Čekej na dokončení (5–10 minut).

Po zápisu vlož SD kartu do RPi, zapoj Ethernet a napájení. Nechej RPi nastartovat – čekej 30–60 sekund. Zelená LED bude rychle blikat při bootování a pak se uklidní.

4
USB boot je aktivován – zelená LED 4× zabliká

Raspberry Pi OS Lite při prvním bootu automaticky nastaví OTP bit pro USB boot. Poznáš to tak, že po cca 30 sekundách zelená LED rychle 4× zabliká – to znamená "USB boot enabled".

Odpoj napájení a vyjmi SD kartu. Od teď bude RPi hledat bootovací zařízení na USB portech, pokud není vložena SD karta.

💡 OTP bit je trvalý

Změna je nevratná – jednou aktivovaný USB boot nelze deaktivovat. Ale to je v pořádku: RPi bude bootovat z SD karty pokud je vložena, jinak z USB. Logika zůstane zachována.

Co dělat pokud nemáš SD kartu

Nemáš po ruce SD kartu? Dvě alternativy:

  • Možnost 1 – Rovnou zkus USB boot: Novější kusy RPi 3 B+ (výroba 2019 a novější) mají USB boot povolený z výroby. Nahraj HAOS přímo na USB SSD (Část 2) a zkus nastartovat bez SD karty. Pokud to funguje – máš hotovo, SD kartu nepotřebuješ.
  • Možnost 2 – Půjčit SD kartu: Stačí libovolná microSD karta od 2 GB. Zapůjč od kolegy, z fotoaparátu nebo stará karta z telefonu.
ℹ️ Jak poznám rok výroby RPi?

Na spodní straně desky RPi je vytištěn text ve formátu Raspberry Pi 3 Model B+ V1.3 a datum výroby (např. 2019). Výroba od roku 2019 = USB boot pravděpodobně povolený z výroby. Zkus prostě přímo nastartovat z USB SSD.

Část 2 – Nahrání Home Assistant OS na USB SSD

Teď nahrajeme samotný Home Assistant OS na mSATA USB SSD. Toto je skutečný bootovací disk systému.

🔴 Důležité upozornění

V Raspberry Pi Imager vybereš USB SSD jako úložiště. Všechna stávající data na SSD budou nenávratně smazána. Ujisti se, že vybíráš správné zařízení – zkontroluj název a velikost. Nikdy nevyber systémový disk Windows!

5
Připoj mSATA USB SSD k počítači

Zasuň mSATA SSD do USB pouzdra, připoj pouzdro k PC přes USB. Windows ho detekuje jako removable disk. Pokud se otevře průzkumník – zavři ho, disk bude přeformátován.

6
Otevři Raspberry Pi Imager a zvol zařízení

Spusť Raspberry Pi Imager. Klikni CHOOSE DEVICE → vyber Raspberry Pi 3.

7
Vyber Home Assistant OS jako operační systém

Klikni CHOOSE OS a naviguj:

  • → Other specific-purpose OS
  • → Home assistants and home automation
  • → Home Assistant
  • → Home Assistant OS X.X (RPi 3/3+/3B+) – vyber nejnovější verzi

Image se automaticky stáhne z internetu (cca 300–500 MB). Nebo si ho stáhni ručně z github.com/home-assistant/operating-system/releases (hledej haos_rpi3-X.XX.img.xz) a použij "Use custom".

8
Vyber USB SSD jako cílové úložiště

Klikni CHOOSE STORAGE → vyber svůj USB SSD disk.

Zkontroluj název a velikost! Typicky se bude jmenovat nějak jako "USB3.0 DISK" nebo název výrobce pouzdra. Pokud vidíš "Windows (C:)" nebo podobně – NEKLIKEJ NA TO!

9
Zapiš image na SSD

Klikni NEXT. Imager se zeptá na OS customisation – klikni NO (HA konfigurujeme přes webové rozhraní, ne zde).

Potvrď smazání: YES. Zápis začne – ukazatel průběhu zobrazí stav. NEODPOJUJ SSD!

  • Fáze zapisování: cca 3–8 minut (dle rychlosti USB)
  • Fáze ověřování: cca 2–5 minut

Po dokončení se zobrazí "Write Successful". Klikni Continue → zavři Imager.

10
Bezpečně odstraň USB SSD

Pravé tlačítko na ikoně USB v System Tray (u hodin) → "Safely Remove Hardware" → vyber SSD → Eject.

Počkej na potvrzení "Safe to Remove" a teprve pak fyzicky odpoj USB SSD od PC.

Časté situace při nahrávání na Windows

Windows chce naformátovat disk po dokončení

Po úspěšném zápisu HAOS image Windows zobrazí dialog "Je nutné naformátovat disk... před použitím". Klikni na CANCEL / Zrušit! Disk obsahuje Linux oddíly které Windows nerozezná, ale jsou správně zapsány. Neformátuj ho!

ℹ️ Proč Windows nabízí formátování?

HAOS používá Linux souborové systémy (ext4, FAT32 pro boot oddíl). Windows umí číst pouze FAT32 a NTFS – ostatní oddíly mu připadají "poškozené". Je to normální. Disk je správně zapsán a RPi ho bez problémů přečte.

Imager hlásí chybu při ověřování

Pokud se ověřování nezdaří, pravděpodobně:

  • Problémový USB kabel nebo port – zkus jiný USB port (preferuj USB 3.0, modrý)
  • Vadné USB pouzdro – otestuj jiným
  • Vadný SSD – vzácné, ale možné

Zkus znovu – Imager sám nabídne opakování.

SSD se v Imager nezobrazuje

  • Odpoj a znovu zapoj USB kabel
  • Zkus jiný USB port (přímo na PC, ne přes hub)
  • Zkontroluj ve Správci zařízení (Device Manager) jestli se disk zobrazuje

Velikost oddílů a automatické rozšíření

HAOS při prvním bootu automaticky rozšíří datový oddíl na celou dostupnou kapacitu SSD. Nemusíš nic dělat ručně.

  • 32 GB SSD: cca 28 GB využitelných pro data HA po instalaci
  • 64 GB SSD: cca 60 GB – doporučeno pokud plánuješ ukládat záznamy kamer nebo necháš databázi růst déle
  • 128 GB SSD: výrazný nadbytek pro základní HA, ale levné mSATA disky tuto kapacitu nabízejí

Struktura oddílů po instalaci (viditelná přes SSH příkazem lsblk):

  • /dev/sda1 – Boot oddíl (FAT32, ~100 MB) – jediný čitelný z Windows
  • /dev/sda2 – HAOS systémový oddíl A (ext4)
  • /dev/sda3 – HAOS systémový oddíl B (záloha pro rollback)
  • /dev/sda8 – Datový oddíl (ext4, zbytek disku) – zde jsou konfigurace, zálohy, databáze

Shrnutí – co teď máš připravené

  • ✅ RPi 3 B+ s aktivovaným USB boot (OTP bit)
  • ✅ mSATA USB SSD s nahraným Home Assistant OS image
  • ✅ Vše připravené pro fyzické zapojení a první boot
