Bezpečnost

DŮLEŽITÉ: Home Assistant by měl být dostupný POUZE z lokální sítě, pokud nevíš přesně co děláš s vzdáleným přístupem! ------------------------------------------------------------------------ 11.1 FIREWALL NA ROUTERU (NEJDŮLEŽITĚJŠÍ OCHRANA) ------------------------------------------------------------------------ PRAVIDLO Č. 1: NIKDY neforwarduj port 8123 (ani žádný jiný HA port) z internetu na RPi! Jak zkontrolovat nastavení routeru: KROK 53: Přihlaš se do routeru (obvykle 192.168.1.1) KROK 54: Zkontroluj sekci "Port Forwarding" / "NAT" / "Virtual Server" -> NESMÍ existovat žádné pravidlo forwardující port 8123 na IP RPi -> Pokud existuje: SMAŽ HO IHNED KROK 55: Zkontroluj sekci "UPnP" / "Universal Plug and Play" -> UPnP může automaticky otevírat porty! -> Doporučuji VYPNOUT UPnP v routeru pro větší kontrolu ------------------------------------------------------------------------ 11.2 FIREWALL V HOME ASSISTANT OS ------------------------------------------------------------------------ HAOS má základní firewall, ale je vhodné přidat pravidla: KROK 56: Nastavení -> Systém -> Host -> Zde lze vidět firewall nastavení Přes SSH (volitelné – nutný SSH addon): Povolit přístup jen z lokální sítě (příklad pro síť 192.168.1.0/24): ha network update eth0 --ipv4-method static --ipv4-address 192.168.1.50/24 ------------------------------------------------------------------------ 11.3 SILNÉ HESLO A DVOUFAKTOROVÁ AUTENTIZACE (2FA) ------------------------------------------------------------------------ KROK 57: Nastav 2FA pro svůj HA účet: -> Klikni na své jméno vlevo dole -> Sekce "Zabezpečení" nebo "Multi-factor Authentication" -> Klikni "Aktivovat" u TOTP (Time-based One-Time Password) -> Naskenuj QR kód aplikací Google Authenticator nebo Authy -> Zapiš si záložní kódy na BEZPEČNÉ místo! -> Klikni "Aktivovat" Po zapnutí 2FA budeš při přihlášení potřebovat kód z mobilu. 2FA PLATÍ POUZE PRO PŘIHLÁŠENÍ – přes lokální síť je to extra ochrana. ------------------------------------------------------------------------ 11.4 TRUSTED NETWORKS – BEZPROBLÉMOVÝ LOKÁLNÍ PŘÍSTUP ------------------------------------------------------------------------ Pokud chceš, aby zařízení v lokální síti nemusela zadávat heslo: KROK 58: V Home Assistant -> Nastavení -> Lidé -> záložka "Poskytovatelé auth" -> Nebo uprav configuration.yaml: Přidej do /config/configuration.yaml: homeassistant: auth_providers: - type: trusted_networks trusted_networks: - 192.168.1.0/24 # tvoje lokální síť - 127.0.0.1 allow_bypass_login: true # bez hesla z lokální sítě - type: homeassistant # standardní přihlášení (záloha) !!! POZOR: Tento přístup je vhodný POUZE pokud je tvoje WiFi/LAN zabezpečena a nikdo cizí se k ní nedostane !!! ------------------------------------------------------------------------ 11.5 ZABEZPEČENÍ MQTT ------------------------------------------------------------------------ KROK 59: Ověř, že Mosquitto poslouchá POUZE na lokálních rozhraních: -> Nastavení -> Doplňky -> Mosquitto broker -> KONFIGURACE -> Přidej nebo ověř: customize: active: false logins: - username: mqtt_user password: [tvoje heslo] certfile: fullchain.pem keyfile: privkey.pem Mosquitto v HAOS je ve výchozím nastavení dostupný POUZE uvnitř HA sítě (port 1883 není přístupný z internetu – je to v pořádku). ------------------------------------------------------------------------ 11.6 VZDÁLENÝ PŘÍSTUP – KDYŽ HO PŘECE JEN CHCEŠ ------------------------------------------------------------------------ POKUD chceš přistupovat k HA z internetu, nikdy neotvírej port 8123! Místo toho použij: VARIANTA A – Nabu Casa (Home Assistant Cloud): -> Placená služba (cca 65 Kč/měsíc) od tvůrců HA -> Nejjednodušší a nejbezpečnější vzdálený přístup -> Nastavení: Nastavení -> Home Assistant Cloud -> Vše šifrované, bez portforwardingu VARIANTA B – Tailscale VPN (ZDARMA): -> VPN síť – připojíš se jako bys byl doma -> Nastavení: 1. Registruj se na https://tailscale.com (zdarma) 2. Nastavení -> Doplňky -> ADD-ON STORE -> hledej "Tailscale" 3. Nainstaluj a nastav Tailscale addon v HA 4. Na telefonu/PC nainstaluj Tailscale klienta 5. Přes Tailscale se připojíš k domácí síti odkudkoli -> Přístup k HA: přes Tailscale IP adresu (100.x.x.x) VARIANTA C – WireGuard VPN na routeru: -> Složitější, vyžaduje podporu v routeru (Turris, Mikrotik, Asus Merlin) -> Nejbezpečnější varianta, veškerý provoz šifrován DOPORUČENÍ: Nabu Casa pro jednoduchost, Tailscale pro bezplatný přístup. NIKDY nepoužívej přímý port forwarding bez HTTPS a silného hesla! ------------------------------------------------------------------------ 11.7 LOGOVÁNÍ A SLEDOVÁNÍ PŘIHLÁŠENÍ ------------------------------------------------------------------------ KROK 60: Nastavení -> Systém -> Protokoly -> Sleduj neúspěšné pokusy o přihlášení -> Filtruj dle "login" nebo "failed" KROK 61: Nastav notifikaci při neúspěšném přihlášení (volitelné): -> Automatizace -> Vytvořit automatizaci -> Trigger: "Log entry" s klíčovým slovem "failed login" -> Akce: Odeslat push notifikaci na mobil POZNÁMKY K BEZPEČNOSTI: [zde si pište vlastní poznámky]
