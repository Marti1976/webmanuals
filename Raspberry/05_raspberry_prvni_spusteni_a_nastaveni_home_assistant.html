První spuštění Home Assistant

Fyzické zapojení všech komponent, první boot z USB SSD, nalezení HA v síti a kompletní průvodce prvním nastavením – od vytvoření účtu po statickou IP adresu.

Fyzické zapojení

Pořadí zapojení je důležité. Napájení zapojuj vždy jako poslední – tím se RPi spustí. Vše ostatní musí být připojeno dřív.

Pořadí zapojení
1
💽 mSATA USB SSD
Zasuň do USB portu – doporučuji horní pravý port (USB 2.0, vhodný pro SSD). Bez SD karty v slotu.
2
📶 Zigbee USB koordinátor (SLZB-07)
Přes USB prodlužovací kabel (15–30 cm) do druhého USB portu. Prodlužovák dongle vystrčí od kovové desky a zlepší Zigbee příjem.
3
🌐 Ethernet kabel
Jeden konec do RPi (RJ-45 port na krátké straně desky), druhý do routeru nebo Ethernet switche.
4
⚡ Napájecí zdroj (Micro-USB)
Jako poslední – tím se systém spustí. Používej originální RPi napájecí zdroj 5V/3A.
⚠️ Bez SD karty!

Do slotu pro SD kartu NIC nevkládej. RPi musí bootovat z USB SSD. Pokud by byla vložena SD karta (i prázdná nebo s jiným OS), RPi by se pokusil bootovat z ní a neúspěšně.

První boot – co se děje

Po připojení napájení se spustí první boot. Tento proces trvá výrazně déle než normální start – systém se rozbaluje, konfiguruje a nastavuje. Buď trpělivý.

  • 🟡 0–2 minuty: RPi bootuje HAOS z USB SSD
  • 🟡 2–10 minut: HAOS rozbaluje kontejnery, stahuje supervisor a core komponenty
  • 🟡 10–20 minut: Supervisor iniciuje Home Assistant Core, stahuje aktuální verzi
  • 🟢 Po 10–25 minutách: Systém je připraven
🔴 Nejdůležitější pravidlo prvního bootu

NIKDY nepřerušuj napájení během prvního bootu! Systém zapisuje databázi, konfiguraci a soubory. Přerušení může způsobit poškození dat a budeš muset celý proces opakovat od začátku.

LED indikátory RPi 3 B+ při bootu

Fyzické LED kontrolky na desce RPi prozradí co se děje i bez přístupu k monitoru:

  • Červená LED (PWR): svítí trvale = napájení v pořádku. Nesvítí vůbec = napájecí problém, zkontroluj zdroj.
  • Zelená LED (ACT): rychlé nepravidelné blikání = aktivita SSD, systém načítá data = normální
  • Zelená LED bliká vzorem 4×: USB boot byl právě aktivován (jen při aktivaci z Kapitoly 04)
  • Zelená LED nesvítí vůbec: systém se nespustil nebo SSD nelze přečíst – zkontroluj zapojení
  • Po dokončení bootu: zelená LED bliká pomalu a nepravidelně – systém běží a zpracovává úlohy

Nalezení IP adresy v síti

Adresa homeassistant.local funguje přes mDNS a nemusí fungovat na všech sítích (zejména starší routery). Jako zálohu si zjisti přímou IP adresu:

🌐 Přes webové rozhraní routeru

Přihlaš se do routeru (obvykle 192.168.1.1 nebo 192.168.0.1). Najdi seznam připojených zařízení (DHCP clients / Connected devices). Hledej zařízení s názvem "homeassistant" nebo "hassio". Poznamenej si IP adresu.

🔍 Přes network scanner

Na Windows: stáhni Advanced IP Scanner (zdarma) nebo použij nmap. Spusť sken lokální sítě (např. 192.168.1.0/24) a hledej zařízení s hostname "homeassistant".

Jakmile máš IP adresu (např. 192.168.1.50), přistupuj přes:

http://192.168.1.50:8123

Průvodce prvním nastavením

Po otevření http://homeassistant.local:8123 nebo IP adresy se zobrazí průvodce. Projdeme ho krok za krokem:

⚠️ Uživatelské jméno nelze po vytvoření změnit!

Zvol pečlivě – doporučuji jednoduché, bez diakritiky a bez mezer (např. admin nebo tvoje jméno bez háčků a čárek). Heslo změnit jde kdykoli, uživatelské jméno bohužel ne – bez kompletního resetu systému.

1
Uvítací obrazovka

Zobrazí se "Welcome to Home Assistant". Klikni "Create my smart home".

2
Vytvoření účtu správce

Toto je tvůj hlavní přihlašovací účet – uložit bezpečně!

  • Jméno: tvoje jméno (zobrazovací)
  • Uživatelské jméno: přihlašovací jméno, bez mezer (např. admin)
  • Heslo: min. 12 znaků, kombinace písmen, čísel a symbolů
  • Potvrď heslo: zadej znovu

Toto heslo nelze obnovit bez přístupu k systému. Zapiš si ho do bezpečného místa (password manager).

3
Nastavení domácnosti a polohy

Tato nastavení se používají pro astronomické automatizace (západ/východ slunce, přítomnost doma).

  • Jméno domácnosti: libovolný název (např. "Moje Home")
  • Časové pásmo: Europe/Prague
  • Poloha: klikni na mapu nebo zadej souřadnice – stačí přibližně, město nebo čtvrť
  • Systém jednotek: Metrické
  • Měna: CZK
4
Přeskočit automatickou detekci zařízení

HA nabídne automaticky nalezená zařízení v síti (Philips Hue Bridge, Sonos, Chromecast...). Tato stránka je volitelná – klikni "Skip" (Přeskočit). Zařízení přidáme manuálně přes Zigbee2MQTT.

5
Dokončení průvodce

Klikni "Finish". Jsi přihlášen do Home Assistant! Uvidíš hlavní dashboard (Overview).

Výchozí dashboard je prázdný – to je v pořádku. Karty a widgety přidáme po nastavení Zigbee zařízení.

Nastavení pevné IP adresy (doporučeno)

DHCP router přiděluje IP adresy dynamicky – po restartu se může IP RPi změnit. Pro server který běží 24/7 chceme pevnou IP, aby záložky v prohlížeči vždy fungovaly.

Varianta A – Rezervace v routeru (nejlepší způsob)

Přidej do routeru DHCP rezervaci: Router vždy přidělí konkrétní IP konkrétní MAC adrese RPi. Výhoda: nakonfigurováno v jednom místě, funguje i po přeinstalaci HAOS.

  1. Přihlaš se do routeru (obvykle 192.168.1.1)
  2. Najdi seznam DHCP klientů – hledej "homeassistant"
  3. Zkopíruj MAC adresu RPi (formát: b8:27:eb:xx:xx:xx)
  4. Najdi sekci DHCP Reservations / Static IP / IP Binding (název závisí na routeru)
  5. Přidej rezervaci: MAC adresa → zvolená IP (např. 192.168.1.50)
  6. Ulož a restartuj router
💡 Jakou IP zvolit?

Zvol IP mimo DHCP rozsah routeru. Zjistíš ho v nastavení routeru (obvykle 192.168.1.100–192.168.1.254). Dobrá volba: 192.168.1.50 nebo 192.168.1.10 – tyto adresy DHCP obvykle nepřiděluje.

Varianta B – Nastavení přímo v Home Assistant

Pokud nemáš přístup do routeru nebo chceš nastavit IP přímo v HAOS:

  1. Jdi do Nastavení → Systém → Síť
  2. Klikni na rozhraní eth0 (Ethernet)
  3. Přepni z "Automatic (DHCP)" na "Static"
  4. Vyplň:
    • IP adresa: 192.168.1.50/24 (nebo dle tvé sítě)
    • Brána (Gateway): 192.168.1.1 (adresa routeru)
    • DNS: 192.168.1.1 (router) nebo 1.1.1.1 (Cloudflare)
  5. Ulož – RPi se restartuje, od té doby má pevnou IP
ℹ️ Po nastavení statické IP

Aktualizuj záložku v prohlížeči na novou adresu (např. http://192.168.1.50:8123). Přístup přes hostname homeassistant.local:8123 bude nadále fungovat.

Nastavení WiFi (alternativa k Ethernet)

Pokud nemůžeš přivést Ethernet kabel, Home Assistant OS podporuje WiFi připojení. Konfiguruje se po prvním spuštění přímo v rozhraní HA.

  1. Jdi do Nastavení → Systém → Síť
  2. Klikni na + Přidat síť nebo na bezdrátové rozhraní wlan0
  3. Zadej SSID (název WiFi sítě) a heslo
  4. Ulož – RPi se připojí k WiFi
  5. Po připojení k WiFi lze Ethernet kabel odpojit
⚠️ Ethernet je vždy lepší volba

Pro server běžící 24/7 doporučuji Ethernet. WiFi může mít výpadky, vyšší latenci a problémy při rušení sítě. HA zapisuje data průběžně – nestabilní spojení způsobuje chyby v databázi. Ethernet kabel = žádné starosti.
