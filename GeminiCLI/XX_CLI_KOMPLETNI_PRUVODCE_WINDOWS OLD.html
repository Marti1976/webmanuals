Co je Gemini CLI a proč ho chceš

Gemini CLI je AI asistent od Googlu, který pracuje přímo v tvém terminálu (příkazovém řádku). To zní technicky, ale za tím se skrývá jednoduchá myšlenka:

Místo toho, abys musel kopírovat kód do webového rozhraní, AI vidí celý tvůj projekt přímo na tvém počítači. Ty mu jen napíšeš česky co chceš, a on to udělá – přečte soubory, opraví chybu, přidá funkci, vysvětlí co kde je.

Co konkrétně Gemini CLI umí

  • Číst celý projekt najednou – všechny soubory, strukturu složek, konfiguraci
  • Upravovat soubory – přidávat kód, opravovat chyby, přejmenovávat věci
  • Vysvětlovat kód – "co tato funkce dělá?", "proč mi to nefunguje?"
  • Psát nové funkce – "přidej tlačítko pro oblíbené recepty"
  • Vyhledávat na internetu – zjistit aktuální řešení problémů
  • Spouštět příkazy – npm install, build, atd. – ale vždy se tě zeptá
  • Psát dokumentaci – README, komentáře v kódu, popisy

Co je to terminál a jak ho otevřít

Terminál (nebo příkazový řádek) je okno, kde se ovládá počítač textem místo myší. Vypadá to strašidelně, ale ve skutečnosti jde o psaní jednoduchých příkazů.

💡
Nemusíš rozumět Linuxu ani být hacker.

Budeš psát maximálně 3–4 různé příkazy. Zbytek času budeš psát česky Gemini co chceš.

Jak otevřít terminál podle systému

Způsob 1 – nejjednodušší:

Stiskni klávesy Win + R, napiš powershell a stiskni Enter.

Způsob 2 – přes Start menu:

Klikni na Start → hledej "PowerShell" nebo "Windows Terminal" → klikni pravým tlačítkem → Spustit jako správce (ne vždy nutné, ale občas pomůže)

Způsob 3 – přímo ve složce projektu:

Otevři složku projektu v Průzkumníku → klikni do adresního řádku (kde je cesta) → smaž text → napiš powershell → Enter. Terminál se otevře rovnou ve správné složce! 🎉

Způsob 1 – Spotlight:

Stiskni Cmd + Mezerník, napiš "Terminal" a stiskni Enter.

Způsob 2 – Finder:

Aplikace → Nástroje → Terminál

💡
Tip pro Mac uživatele

Doporučuji stáhnout iTerm2 (zdarma) – je lepší než výchozí terminál. Ale není nutný.

Stiskni Ctrl + Alt + T nebo hledej "Terminál" v aplikacích.

Pokud čteš tuto sekci, pravděpodobně to víš lépe než já. 😄

Základní příkazy v terminálu – stačí tyhle 3

Příkazy pro pohyb

cd ..Přejdi o složku VÝŠE (dva tečky = nahoru)

V terminálu stačí stisknout šipku ↑ a opakuješ předchozí příkazy. Nemusíš je psát znova.

Instalace Node.js (nutný základ)

Gemini CLI je napsané v JavaScriptu a potřebuje Node.js – prostředí pro spouštění JavaScriptu. Naštěstí to tvůj projekt již pravděpodobně má, ale pojďme si to ověřit.

Krok 1 – Ověř zda Node.js máš

Otevři terminál a napiš:

# Zkontroluj verzi Node.js: node --version # Zkontroluj verzi npm (správce balíčků): npm --version

✅ Tohle je dobrý výsledek

Vidíš číslo jako v20.11.0 nebo vyšší → Node.js máš a funguje. Přeskoč rovnou na sekci 5.

❌ Tohle je problém

Vidíš command not found nebo verzi nižší než 20 → musíš nainstalovat nebo aktualizovat.

Krok 2 – Instalace Node.js (pokud chybí nebo je starý)

  1. Jdi na https://nodejs.org
  2. Stáhni verzi označenou LTS (vlevo – ta stabilní)
  3. Spusť stažený instalátor – vše klikni "Next", nechej výchozí nastavení
  4. Po instalaci zavři a znovu otevři terminál (důležité!)
  5. Znovu zadej node --version – teď by to mělo fungovat
⚠️
Nezapomeň restartovat terminál!

Windows si nové programy "nevšimne" dokud terminál nezavřeš a neotevřeš znovu.

Způsob 1 – přes web (nejjednodušší):

  1. Jdi na https://nodejs.org
  2. Stáhni verzi LTS
  3. Spusť .pkg instalátor, klikej Next

Způsob 2 – přes Homebrew (pro zkušenější):

brew install node

Instalace Gemini CLI

Tohle je ta jednoduchá část. Jeden příkaz a je hotovo.

1
Otevři terminál a napiš tento příkaz
npm install -g @google/gemini-cli

Příznak -g znamená "nainstaluj globálně" – takže příkaz gemini bude fungovat odkudkoliv.

Instalace trvá 1–3 minuty. Uvidíš hodně textu – to je normální. Čekej než se vrátí řádek s výzvou.

2
Ověř že instalace proběhla
gemini --version

Měl bys vidět číslo verze. Pokud vidíš command not found, zkus zavřít a znovu otevřít terminál.

⚠️
Chyba "EACCES: permission denied" na Macu?

Zkus příkaz s sudo před ním: sudo npm install -g @google/gemini-cli Zadáš heslo k Macu (při psaní se nic nezobrazuje – to je normální) a zmáčkneš Enter.

Vývojový server – jak vidět změny okamžitě

Tvůj receptář je React aplikace. Nemůžeš ji jednoduše otevřít jako HTML soubor v prohlížeči – potřebuje být zkompilována za běhu. To dělá vývojový server. A nemusíš nic nahrávat na GitHub – vše testuje lokálně na svém počítači.

Vývojový server = mini-hosting přímo na tvém PC.

Spustíš ho jedním příkazem a aplikace je dostupná na adrese http://localhost:3000 – jen pro tebe, nikdo jiný ji nevidí. A při každé změně souboru se stránka obnoví automaticky.

Jak spustit vývojový server

1
V terminálu přejdi do projektu a spusť server
# Přejdi do složky projektu: cd cesta/k/rodinny-receptar # Spusť vývojový server: npm run dev

Uvidíš výstup podobný tomuto:

Terminál 1 – dev server
$ npm run dev VITE v6.2.0 ready in 312 ms ➜ Local: http://localhost:3000/ ➜ Network: http://192.168.1.105:3000/
2
Otevři aplikaci v prohlížeči

Otevři Chrome a jdi na adresu http://localhost:3000. Měl bys vidět svůj receptář živě.

localhost = tvůj počítač  |  3000 = číslo portu (jako číslo dveří)

Pokud 3000 nefunguje, zkus http://localhost:5173 – Vite někdy přepne port automaticky.

3
Nechej server běžet – nezavírej tento terminál!
⚠️
Terminál s npm run dev musí zůstat otevřený celou dobu.

Pokud ho zavřeš, aplikace v prohlížeči přestane fungovat. Spustíš ho jednou a pak na něj zapomeneš.

HMR – proč se stránka obnovuje sama

Vite má zabudovanou funkci zvanou HMR (Hot Module Replacement). Sleduje tvé soubory a při každé změně automaticky aktualizuje jen tu část stránky, která se změnila – bez F5, bez restartu.

Terminál 1 – co uvidíš když Gemini změní soubor
➜ Local: http://localhost:3000/ <-- čekáš, Gemini mezitím mění soubor v Terminálu 2 --> [vite] hmr update /src/components/RecipeCard.tsx <-- prohlížeč se automaticky obnoví do 1-2 sekund -->

Kdy je potřeba server restartovat (ručně)

✅ HMR funguje automaticky pro

Změny v komponentách (.tsx), styly, text, logika – vše v src/ složce.

⚠️ Potřeba restartu serveru

Změny v vite.config.ts, package.json, nebo instalace nových balíčků (npm install). Stiskni Ctrl+C a znovu npm run dev.

Soubor GEMINI.md – klíč ke všemu

Toto je nejdůležitější věc celého průvodce. Soubor GEMINI.md v kořenové složce projektu Gemini CLI automaticky přečte při každém spuštění. Je to jako "briefing" pro AI – řekneš mu kdo jsi, co dělá projekt, a hlavně co smí a co nesmí dělat.

🔴
Bez GEMINI.md = Google AI Studio zážitky se opakují

AI neví co je důležité, co nesmí smazat, jak má postupovat. S tímto souborem mu dáš jasná pravidla, která platí celou dobu.

Jak soubor vytvořit

V kořenové složce projektu (tam kde je package.json) vytvoř nový textový soubor s názvem GEMINI.md. Lze to přes VS Code, Notepad, nebo klidně přes Průzkumník (nový textový soubor → přejmenuj na GEMINI.md).

Zkopíruj tento obsah a uprav si ho dle sebe:

GEMINI.md — umísti do kořenové složky projektu

Nejdřív mi ŘEKNI co hodláš udělat a proč 2. Počkej na mé výslovné potvrzení (ANO/OK) 3. Zálohy – NIKDY nepřeskoč

🔴
AI může udělat chybu. Vždy. I s pravidly v GEMINI.md.

Záloha = 2 minuty práce. Obnova bez zálohy = hodiny nebo ztracená práce navždy. Volba je jasná.

Možnost A – Git (doporučeno, ale vyžaduje nastavení)

Git je systém pro sledování změn v kódu. Každý "commit" je snapshot projektu ke kterému se lze vrátit. Tvůj projekt ho pravděpodobně již má (pokud vidíš složku .git).

# Před každou prací s Gemini udělej zálohu (commit): git add . git commit -m "záloha před Gemini – datum" # Pokud Gemini udělá blbost – vrátíš se ke commit: git checkout . # Toto vrátí VŠECHNY soubory do stavu posledního commitu # Zobraz historii záložních bodů: git log --oneline

Možnost B – Ruční záloha složky (pro jistotu)

Jednoduché, funguje vždy, nevyžaduje znalost Gitu:

  1. Najdi složku projektu v Průzkumníku
  2. Klikni pravým tlačítkem → Kopírovat
  3. Vložit někam jinam (plocha, jiná složka)
  4. Přejmenuj zálohu na rodinny-receptar-zaloha-DATUM
💡
Ideální combo: Git commit + záloha složky

Git pro rychlé vrácení změn, záloha složky jako pojistka pro případ, že se něco pokazí i s Gitem.

Co říct Gemini na začátku každé práce

Před každou změnou v souboru mi nejdřív řekni co chceš změnit a proč. Dělej vždy jen jednu věc najednou a počkej na mé potvrzení. Pokud si nejsi jistý, raději se zeptej.

Když to nefunguje – řešení běžných problémů

!
❌ "gemini: command not found"

Příčina: Gemini CLI není nainstalován nebo ho terminál nenachází.

Řešení 1: Zavři terminál a otevři znovu (Windows si nové příkazy načte až po restartu terminálu).

Řešení 2: Znovu spusť instalaci: npm install -g @google/gemini-cli

Řešení 3 (Mac): Zkus npx @google/gemini-cli místo jen gemini

!
❌ "node: command not found" nebo stará verze

Příčina: Node.js není nainstalován nebo je starý (potřeba 20+).

Řešení: Stáhni a nainstaluj Node.js LTS z nodejs.org. Po instalaci znovu otevři terminál.

!
❌ "EACCES: permission denied" při instalaci (Mac)

Řešení: Přidej sudo před příkaz:

sudo npm install -g @google/gemini-cli

Zadáš heslo k Macu – při psaní se nic nezobrazuje, ale zadává se. Stiskni Enter.

!
❌ Přihlášení nefunguje / prohlížeč se neotevřel

Řešení 1: Zkus příkaz gemini auth login

Řešení 2: Zkontroluj zda máš funkční připojení k internetu.

Řešení 3: Pokud se prohlížeč neotevřel automaticky, v terminálu bude URL odkaz – zkopíruj ho a vlož ručně do prohlížeče.

?
⚠️ Gemini "nevidí" soubory projektu

Příčina: Spustil jsi Gemini CLI v jiné složce, ne ve složce projektu.

Řešení: Ukonči Gemini (/quit), přejdi do správné složky pomocí cd, a znovu spusť gemini.

Ověř kde jsi: pwd (Mac/Linux) nebo cd (Windows bez argumentu)

?
⚠️ Gemini ignoruje pravidla z GEMINI.md

Příčina 1: Soubor GEMINI.md je ve špatné složce. Musí být přímo tam, kde spouštíš Gemini (kořen projektu).

Příčina 2: Překlep v názvu souboru – musí být přesně GEMINI.md (velká písmena)

Řešení: Ujisti se, že soubor je vedle package.json. Ne ve složce, ale vedle ní.

?
⚠️ Dosáhl jsi denního limitu (1000 dotazů)

To se při hobby projektu stane těžko, ale pokud ano – limit se obnoví druhý den o půlnoci (UTC čas).

Pokud to nastane opakovaně, pracuješ fakt hodně. 😄
