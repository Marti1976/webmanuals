Co je vývojový server a proč ho potřebuješ

Tvůj receptář je React aplikace. React aplikace nemůžeš prostě otevřít jako HTML soubor v prohlížeči – vyžaduje kompilaci, zpracování TypeScriptu, Tailwindu a dalších věcí. To za tebe dělá vývojový server.

Vývojový server = lokální "mini-hosting" na tvém počítači.

Spustíš ho jedním příkazem a aplikace je dostupná na adrese http://localhost:3000. To je tvá aplikace, běžící jen na tvém počítači, nikdo jiný ji nevidí.

Tvůj projekt používá Vite

Vite je moderní nástroj pro vývoj webových aplikací. Pro tebe to znamená jednu klíčovou věc:

✏️
Gemini změní soubor např. přidá tlačítko
⚡
🌐
Prohlížeč se obnoví do 1–2 sekund, sám

Nemusíš nic mačkat, nic restartovat, nic nahrávat. Prostě koukáš do prohlížeče a vidíš změny okamžitě.

💡
Tohle se jmenuje HMR – Hot Module Replacement

Vite sleduje tvé soubory a při každé změně okamžitě aktualizuje jen tu část stránky, která se změnila. Ani neztratíš stav aplikace (co máš rozepsané, kde jsi scrolloval).

// sekce 02

Jak spustit vývojový server

1
Otevři terminál a přejdi do projektu

Musíš být ve složce projektu (tam kde je package.json).

# Přejdi do složky projektu: cd C:\Users\Pavel\projekty\rodinny-receptar # Ověř že jsi na správném místě (měl bys vidět package.json): ls ← Mac/Linux dir ← Windows
2
Spusť dev server
npm run dev

Vite se spustí a uvidíš výstup podobný tomuto:

Terminal – dev server
$ npm run dev VITE v6.2.0 ready in 312 ms ➜ Local: http://localhost:3000/ ➜ Network: http://192.168.1.105:3000/ ➜ press h + enter to show help
3
Otevři aplikaci v prohlížeči

Otevři prohlížeč (Chrome doporučeno) a jdi na adresu:

Adresa aplikace
localhost:3000

localhost = tvůj počítač (ne internet)

3000 = port (číslo "dveří" ke tvé aplikaci)

Případně zkus http://localhost:5173 pokud 3000 nefunguje – Vite někdy používá jiný port.

4
Nechej server běžet

Terminál s dev serverem nezavírej a nepřerušuj. Musí běžet celou dobu co pracuješ. Pokud terminál zavřeš, aplikace v prohlížeči přestane fungovat.

⚠️
Terminál s dev serverem = nechej otevřený

Spustíš ho jednou a zapomeneš na něj. Veškerou práci dělat budeš ve druhém terminálu s Gemini CLI.

// sekce 03

Dva terminály vedle sebe – nastavení

Tohle je klíč k celému pracovnímu postupu. Potřebuješ dvě okna terminálu otevřená zároveň.

Jak to vypadá na obrazovce
Terminál 1 npm run dev
← nechej běžet
+
Terminál 2 gemini
← tady pracuješ
+
Prohlížeč localhost:3000
← vidíš výsledek

Jak otevřít druhý terminál

🪟 Windows – PowerShell

Klikni pravým tlačítkem na ikonu PowerShell na hlavním panelu → "Otevřít nové okno". Nebo Win+R → powershell → Enter.

🪟 Windows Terminal (moderní)

Klikni na šipku dolů vedle záložky → "New tab" nebo Ctrl+Shift+T. Oba terminály jsou pak jako záložky.

🍎 Mac – iTerm2

Cmd+T = nová záložka, nebo Cmd+D = rozděl okno vedle sebe. Doporučuji rozdělit vedle sebe.

🍎 Mac – výchozí Terminal

Shell → Nové okno → Nové okno. Nebo Cmd+N = nové okno terminálu.

💡
Tip: VS Code má integrovaný terminál

Pokud používáš VS Code pro prohlížení kódu, má vlastní terminál dole. Ctrl+` (backtick) ho otevře. Můžeš mít terminály jako záložky přímo ve VS Code.

Kompletní setup – co a kde spustit

// sekce 04

HMR – jak funguje automatické obnovení

HMR (Hot Module Replacement) je funkce Vite, která sleduje změny v souborech a okamžitě je promítne do prohlížeče. Nemusíš mačkat F5.

Co se stane krok za krokem

1
Gemini upraví soubor (např. RecipeCard.tsx)

Ve druhém terminálu Gemini napíše nový kód a uloží soubor na disk.

2
Vite si toho okamžitě všimne

V prvním terminálu (kde běží dev server) uvidíš zprávu:

[vite] hmr update /src/components/RecipeCard.tsx
3
Prohlížeč se sám obnoví

Do 1–2 sekund vidíš změnu v prohlížeči. Žádné F5, žádný restart.

✅ Kdy HMR funguje perfektně

Změny v komponentách, styly, text, logika – vše co je uvnitř src/ složky.

⚠️ Kdy je potřeba restart serveru

Změny v vite.config.ts, package.json, nebo instalace nových npm balíčků. Pak stiskni Ctrl+C v prvním terminálu a znovu npm run dev.

// sekce 05

Všechny npm příkazy pro tvůj projekt

Npm je správce balíčků – nástroj který spravuje všechny závislosti projektu. Příkazy npm run ... jsou definované v package.json tvého projektu.

PříkazCo děláKdy použít
npm run dev Spustí vývojový server na localhost:3000 Vždy při práci na projektu – hlavní příkaz
npm run build Vytvoří produkční verzi do složky /dist Před nahráním na hosting/GitHub Pages
npm run preview Spustí preview produkčního buildu lokálně Ověření buildu před nasazením
npm install Nainstaluje všechny závislosti ze package.json Po stažení projektu z Gitu, nebo po přidání nových balíčků
npm install název-balíčku Přidá nový balíček do projektu Když potřebuješ novou knihovnu (vždy se ptej Gemini)
npm run lint Zkontroluje kód na chyby (TypeScript check) Před odevzdáním/nasazením kódu
npm run clean Smaže složku /dist Když chceš čistý build od začátku
⚠️
Po Ctrl+C musíš znovu spustit server

Pokud omylem stiskneš Ctrl+C v terminálu s dev serverem, server se zastaví. Stačí znovu napsat npm run dev a spustit ho znovu.

Jak zastavit dev server

# V terminálu kde běží npm run dev stiskni: Ctrl + C # Server se zastaví, terminál je zase volný # Znovu spustit: npm run dev

Kompletní pracovní postup od A do Z

Tohle je přesně to co budeš dělat každý den kdy budeš pracovat na receptáři.

1
Udělej zálohu (git commit)
git add . git commit -m "záloha před prací $(date)"

Vždy jako první. Nikdy nevíš co AI udělá.

2
Otevři dva terminály, spusť dev server v Terminálu 1
── TERMINÁL 1 ── cd cesta/k/projektu npm run dev → čekej až uvidíš "Local: http://localhost:3000/"
3
Otevři aplikaci v prohlížeči

Chrome nebo Firefox → adresa http://localhost:3000 → Enter.

Měl bys vidět svůj receptář. Nechej toto okno otevřené.

4
Spusť Gemini CLI v Terminálu 2
── TERMINÁL 2 ── cd cesta/k/projektu gemini Gemini > Přečti GEMINI.md a čekej na mé pokyny. Pracuj česky.
5
Pracuj – piš instrukce Gemini, sleduj prohlížeč
Gemini > Přidej do RecipeCard tlačítko pro sdílení receptu Gemini: OK, přidám tlačítko. Upravím soubor RecipeCard.tsx, řádek 34. Smím to udělat? Gemini > Ano ✅ Hotovo. Podívej se do prohlížeče.

V prohlížeči uvidíš změnu automaticky do 1–2 sekund.

6
Konec práce – ulož a zastav
V Terminálu 2 (Gemini): /quit V Terminálu 1 (dev server): Ctrl + C Git commit všeho co jsi udělal: git add . git commit -m "přidáno tlačítko sdílení"
// sekce 07

Problémy s dev serverem – řešení

!
❌ Prohlížeč hlásí "This site can't be reached"

Dev server není spuštěný. Jdi do Terminálu 1 a zkontroluj jestli tam běží npm run dev. Pokud ne, spusť ho znovu.

!
❌ Port 3000 je already in use
Error: Port 3000 is already in use

Jiná instance dev serveru již běží (nebo jiný program). Buď ji najdi a zastav (Ctrl+C v jiném terminálu), nebo změň port:

npm run dev -- --port 3001

Pak otevři http://localhost:3001

!
❌ Prohlížeč neukazuje změny (HMR nefunguje)

Zkus tyto věci v pořadí:

  1. Tvrdé obnovení stránky: Ctrl+Shift+R (nebo Cmd+Shift+R na Mac)
  2. V Terminálu 1 zastav server (Ctrl+C) a znovu spusť npm run dev
  3. Vymaž cache prohlížeče: DevTools → Application → Storage → Clear site data
!
❌ Chyba při spuštění npm run dev
Error: Cannot find module '...'

Pravděpodobně chybí node_modules složka. Spusť:

npm install

Počkej až se dokončí, pak znovu npm run dev.

?
⚠️ Gemini přidal balíček, server se rozbil

Gemini mohl přidat závislost do package.json ale neinstaloval ji. Zastav server a spusť:

npm install npm run dev
// sekce 08

Adresy a porty – co znamenají

🏠 localhost:3000

localhost = tvůj počítač. Nikdo jiný tuto adresu nevidí. Je to jako "doma".

3000 = číslo portu. Tvůj projekt je nakonfigurován na tento port v souboru vite.config.

🌐 192.168.x.x:3000

Tato adresa (Network) funguje na zařízeních ve stejné WiFi síti. Můžeš si otevřít aplikaci i na mobilu!

Stačí zadat tuto IP adresu do prohlížeče na mobilu (připojeném na stejnou WiFi).

📱
Tip: Testování PWA na mobilu přes WiFi

Tvůj receptář je PWA (dá se nainstalovat v Chrome na mobilu). Při vývoji ho můžeš testovat na mobilu: v terminálu najdi "Network: http://192.168.x.x:3000" a tuto adresu otevři v Chrome na mobilu. Obě zařízení musí být na stejné WiFi.

⚠️
localhost ≠ internet

Aplikace na localhost není dostupná z internetu. Aby ji viděl někdo jiný, musíš ji nasadit na hosting (GitHub Pages, Netlify, atd.) – ale to je až po vývoji.
