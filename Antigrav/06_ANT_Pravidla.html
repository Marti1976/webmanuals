Rules vs Instructions — rozdíl

Instructions (v promptu)

Vždy když napíšeš agentovi "Udělej X" — to je jednoční instrukce. Agent ji udělá a pak čeká na další.

Příklad: jednorázová instrukce

Ty: "Přidej button Logout"
Agent: Dělá
Ty: "Teď CSS"
Agent: Dělá

Rules (v souboru)

Permanentní pravidla která agent používá v KAŽDÉ konverzaci. Jsou to věty jako "VŽDY" nebo "NIKDY".

Příklad: permanentní pravidlo
RULE: Never modify package.json without explicit approval RULE: Always check tests pass before finishing
✓
Rules jsou jako vnitřní instrukcní pro agenta

Ty je jednou napíšeš a agent je követuje navždy, pokud je používáš v tomhle projektu. Perfektní pro osobní hranice.

// sekce 02

ANTIGRAVITY.rules soubor

V kořeni projektu vytvoříš soubor .antigravity-rules (nebo ANTIGRAVITY.rules).

Jak vytvořit

V Antigravity: File Explorer → project root → New File → .antigravity-rules

Obsah je prostý text. Jedna pravidla na řádek.

Struktura

# Komentáře začínají #

RULE: Nikdy neprováduj více změn najednou
RULE: Před každou akcí se mě ptej
ALLOW: modificar soubory v /src
DENY: delete libovolné soubory
CONSTRAINT: Zkontroluj že build projde

Klíčová slova

  • RULE: Obecné pravidlo kterého si agent pamatuje
  • ALLOW: Coagent smí dělat (Allow list)
  • DENY: Co agent nesmí dělat (Deny list)
  • CONSTRAINT: Omezení — co ověřit na konci
  • WORKFLOW: Sekvence akcí (viz sekce 04)
// sekce 03

Syntaxe pravidel — konkrétně

Příklad úplného rules souboru

# Rodinný Receptář — Agent Rules
# Vytvořeno: duben 2026

# === ZÁKLADNÍ PRAVIDLA ===
RULE: Vždy se ptej na schválení než něco mazes
RULE: Menš měnit pouze jeden soubor na úkol
RULE: Pokud selže build, okamžitě řekni proč

# === POVOLENÍ ===
ALLOW: modify soubory v /src
ALLOW: modify soubory v /public
ALLOW: run npm run build
ALLOW: run npm run dev

# === ZÁKAZY ===
DENY: delete jakékoli soubory
DENY: modify package.json
DENY: run npm install
DENY: modify .git

# === OMEZENÍ ===
CONSTRAINT: Všechny soubory TypeScript musí projít TS check
CONSTRAINT: Build musí projít bez warnings
CONSTRAINT: Udělej screenshot výsledku

Bezpečné pravidlo vs bezpečné nebylo

✓ Konkrétní

"DENY: modify soubory mimo /src"

✗ Vágní

"DENY: nedělej blbosti"

// sekce 04

Workflows — sekvence akcí

Pro pokročilé: můžeš definovat celé workflow — sadu akcí které mají být provedeny v pořadí.

Syntaxe WORKFLOW

WORKFLOW: feature-complete
STEP: Vytvoř komponentu
STEP: Naprocvič CSS
STEP: Napiš testy
CONSTRAINT: Všechny testy prochází
CONSTRAINT: Build OK

Jak to funguje?

Když napíšeš agentovi: "Spusť workflow feature-complete" — agent to provede krok za krokem.

ℹ️
Pro tvůj receptář: možná zbytečné

Workflow je pro komplexní projekty. Ty si stačí napsat jednotlivé prompty.

Praktické příklady

Příklad 1: Security rules

# Security
RULE: Nikdy nemaž .env soubory
RULE: Nikdy nemaž .git složku
RULE: Nikdy nemodifikuj soubory mimo /src
DENY: modify .env
DENY: modify .env.local

Příklad 2: Code quality

# Code Quality
CONSTRAINT: Build musí projít bez TS errors
CONSTRAINT: ESLint musí být OK
CONSTRAINT: Uděláš screenshot výsledku
RULE: Přidej JSDoc komentář k nové funkci
RULE: Nepoužívej `any` typ v TypeScript

Příklad 3: Communication

# Communication
RULE: Vždy vysvětli co jsi změnil
RULE: Před smazáním se zeptej
RULE: Při chybě vysvětli proč se to stalo
CONSTRAINT: Odpověz v češtině

Pro tvůj receptář — co napsat

Tady je .antigravity-rules soubor kterej bych doporučil:

# Rodinný Receptář — Agent Rules
# Projekt: rodinny-receptar

# === ZÁKLADNÍ PRAVIDLA ===
RULE: Jedna změna = jeden commit
RULE: Vždy se ptej než smazáš cokoliv
RULE: Vysvetli co se zmenilo a proc

# === POVOLENI ===
ALLOW: modify /src
ALLOW: modify /public
ALLOW: run npm run dev
ALLOW: run npm run build
ALLOW: run npm run test
ALLOW: git commit

# === ZÁKAZY ===
DENY: delete soubory
DENY: modify package.json
DENY: modify .git
DENY: run npm install
DENY: run npm uninstall

# === OMEZENIENIA ===
CONSTRAINT: TypeScript build musí projít
CONSTRAINT: Udělej screenshot výsledku
CONSTRAINT: Všechny změny commituj

Jak to nastavit

  • Vytvoř soubor .antigravity-rules v kořeni projektu
  • Vlož obsah výše
  • Git add + commit: `git add .antigravity-rules && git commit -m "Add agent rules"`
  • Agent to automaticky detekuje a bude to používat
✓
Tímto máš jasně daná pravidla

Agent je bude automaticky dodržovat. Měl by být disciplinovaný bez toho aby jsi musel pokaždé říkat co smí a nesmí.
