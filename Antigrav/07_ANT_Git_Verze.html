Git in Antigravity

Antigravity má vestavěnou Git integraci — podobně jako VS Code. Je to důležité pro tvou bezpečnost.

Co Antigravity dělá s Gitem

  • Vidí tvůj Git status — co se změnilo od posledního commitu
  • Umožňuje ti commitovat (nebo agentovi commitovat)
  • Vidí historii commits (git log)
  • Umožňuje checkout — vrátit se na starší verzi
  • Řeší merge konflikty když se někdo změní

Git vs Antigravity

Git existuje nezávisle — na disku. Antigravity jen s ním komunikuje. Můžeš používat Git z terminálu, VS Code, nebo Antigravity — všechno vidíš.

ℹ️
Bez Gitu se Antigravity také dá

Ale kdyby se agent zbláznil — nemáš jak se vrátit. Git je tvoje pojistka.

// sekce 02

Source Control panel — kde je Git

Levý sidebar v Antigravity → ikona "větvičky" (Source Control)

Co tam vidíš

  • CHANGES — co se změnilo od posledního commitu (soubory s "M", "A", "D")
  • STAGED CHANGES — co jsi vybral na commit
  • COMMIT MESSAGE — pole pro zprávu
  • Branches — jaká větev je aktivní
  • HISTORY — poslední commity

Jak commitovat

1. Vidíš změněné soubory
2. Klikni na "+" vedle souboru → stages se (přesune do STAGED)
3. Napiš zprávu do pole
4. Klikni [Commit] tlačítko
5. Hotovo — změna je v Gitu
// sekce 03

Agent & Git commits — jak to funguje

Scénář 1: Agent změní soubor

Agent udělá změnu → změní se soubor na disku → Git to vidí jako CHANGED

Ty: vidíš v Source Control panelu že se soubor změní

Ty: máš volbu — commitnout (uložit do Gitu) nebo vrátit se git checkout

Scénář 2: Agent chce commitovat sám

Pokud máš v Rules nastaveno ALLOW: git commit, agent si může commitovat sám.

⚠️
To je riziková volba

Pokud agent commituje a později zjistíš že to bylo špatně — musíš znát git revert. Raději ručě commituj.

Scénář 3: Oba (ty i agent) měníte stejný soubor

Ty edituješ RecipeCard.tsx → agent také chce editovat RecipeCard.tsx

Konflikt! Git ti to řekne. Musíš rozhodnutí kdo má pravdu. (Viz sekce 04)

✓
Strategie: Commituj před agentem

Před tím než dáš úkol agentovi, commituj svůj kód. Pak agent má jasnou výchozí pozici a konflikty jsou méně pravděpodobné.

// sekce 04

Konflikty — co když se změní to samé

Jak se konflikt stane

1. Ty jsi commitnul: RecipeCard.tsx v1
2. Agent změnil: RecipeCard.tsx v2 (jinou součást)
3. Git vidí: oba měníte stejný soubor
4. KONFLIKT — kdo má pravdu?

Jak to vypadá v editoru

<<<<<<< HEAD (tvůj kód)
const x = 5;
=======
const x = 10; (agentův kód)
>>>>>>> agent-branch

Jak to vyřešit

Ruční rozlišení: Otevři soubor, rozhodni se který kód chceš, smaž markery.

V Antigravity: Source Control panel má "Resolve Conflicts" — můžeš vybrat "Current" (tvůj) nebo "Incoming" (agent).

Strategie aby to nebyl problém

  • Commit svůj kód — než dáš úkol agentovi
  • Agent pracuje jiné soubory — všichni editují různé věci
  • Sekvence: ty → agent → ty — ne paralelně
⚠️
Pokud je konflikt složitý

Zeptej se agenta: "Vidím konflikt. Co jsi měnil? Jak to vyřešit?" Agent se často sám podvolí tvému kódu.

// sekce 05

Vrácení zpět — undo a revert

Situace: Agent dělal nezvladatelné věci

Řešení: vrátit se na poslední dobrý commit.

Metoda 1: Git Checkout (pokud jsi necommitl)

# Vrátí VŠECHNY změny v souborech
git checkout .

# Nebo konkrétní soubor:
git checkout src/App.tsx

Metoda 2: Git Revert (pokud jsi commitl špatný commit)

# Podívej se na poslední commity:
git log --oneline

# Vrátí konkrétní commit:
git revert HEAD (vrátí poslední commit)
git revert abcd123 (vrátí konkrétní commit)

Metoda 3: V Antigravity GUI

Source Control → History → pravý klik na commit → "Revert to this commit"

✓
Metoda 1 je nejjednoduší

Pokud se agent zbláznil ale necommitl — git checkout . a všechno se vrátí. Jednoduché.

// sekce 06

Strategie pro bezpečné backupy

Golden Rule: Commit před agentem

# Tvůj pracovní postup:
git add . && git commit -m "Before agent work"

# Teď dáš agentovi úkol
# Pokud se zblázní: git reset --hard HEAD~1
# (vrátí se o jeden commit zpět)

Větvení (Branching) — pro větší úkoly

# Vytvoř novou větev:
git checkout -b feature/price-filter

# Agent pracuje v tomhle branch
# Pokud se zblázní: smaž branch
git checkout main
git branch -D feature/price-filter
# Main je v bezpečí

Remote Backup — na GitHub

Pokud máš projekt na GitHubu:

# Push do GitHubu (bezpečný backup)
git push origin main

# Pokud se všechno pokazí:
git reset --hard origin/main
# (vrátí se na to co je na GitHub)
✓
Tady je tvoje pojistka

GitHub = remote backup. Pokud se agent zblázní, tvůj kód na GitHubu pořád existuje. Budeš trvat.

// sekce 07

Pro tvůj receptář — konkretnĺ postup

Nastavení Git v receptáři

Pokud to ještě nemáš:

cd rodinny-receptar
git init (inicializuj git)
git add .
git commit -m "Initial commit"

Pracovní postup s Antigravity

  1. Otevři projekt v Antigravity
  2. Commituj svůj aktuální kód: git add . && git commit -m "Checkpoint before agent work"
  3. Dej agentovi úkol
  4. Agent pracuje a změní soubory
  5. Vidíš změny v Source Control panelu
  6. Pokud jsou OK: commituj — git commit -m "Agent: přidal filtr na cenu"
  7. Pokud nejsou OK: git checkout . a vrátíš se

Emergency Undo

Pokud se agent zbláznil a commitnul špatný kód:

# Vrátí se o jeden commit zpět:
git reset --hard HEAD~1

# Nebo na konkrétní commit:
git log --oneline (podívej se na historii)
git reset --hard abc123 (vrátí se na ten commit)
ℹ️
Tvůj receptář je v bezpečí

Pokud máš Git nastavený a commituješ před agentem — není co se bát. V nejhorším případě git reset a jsi zpět.
