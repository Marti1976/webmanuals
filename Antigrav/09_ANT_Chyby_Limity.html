// sekce 01

Chyby agenta — co může selhati

Scénář 1: Agent nerozumí úkolu

Agent: "Nepochopil jsem co chceš. Můžeš mi vysvětlit jasnĕji?"
Řešení:

Buď konkrétní. Místo "Přidej button" → "Přidej button Delete v RecipeCard.tsx na řádku 42, vedle buttonu Save."

Scénář 2: Agent dělá více než chceš

Ty: "Přidej CSS"
Agent plán: "Přidám CSS, refakturuji logiku, updatnu testy"
Řešení:

V approval kroku: [Edit] → "Jen CSS, nic jiného. Refaktor později."

Scénář 3: Agent smazal co neměl

Agent: "Smazal jsem stare komponenty"
Ty: "To byla komponenta kterou jsi měl zachovat!"
Řešení:

git checkout . nebo git reset --hard HEAD — vrátíš se. V pravidlech: DENY: delete files

// sekce 02

Build chyby — TypeScript a runtime

Chyba 1: TypeScript error

error TS2339: Property 'name' does not exist on type '{}'
Co se stalo:

Agent změnil typ nebo zapomněl property. TypeScript kontrola selha.

Řešení:

Řekni agentovi error message. Agent to obvykle sám opraví.

Chyba 2: Runtime error

Cannot read property 'map' of undefined
at App.tsx:23
Co se stalo:

Agent předpokládá že data existují, ale nejsou tam. Null/undefined problem.

Řešení:

Agent musí přidat null check: data?.map(...) nebo if (data)

Chyba 3: Module not found

Module not found: 'react-dom'
Co se stalo:

Závislost není instalovaná nebo agent importuje špatný balíček.

Řešení:

npm install react-dom a agent to opraví.

// sekce 03

API limity — Gemini quota

Antigravity používá Google Gemini API. Máš limity free tier.

Jaké limity?

⚡

Requesty za minutu (RPM)

Free: 60 requesty za minutu

Pokud agent dělá 10 kroků za minutu — je to limit. Musíš počkat.

📊

Tokeny za den

Free: 50,000 input tokenů / den

Pokud máš velký projekt s dlouhými soubory — můžeš přesáhnout.

🔄

Concurrent requests

Free: 1 request najednou

Manager View s více agenty? Nejde. Musí počkat na sebe.

Chyba: Rate limit exceeded

API Error: Rate limit exceeded. Retry after 60s
Co dělat:

Počkej minutu. Poté zkus znovu. Je to normální — free tier omezení.

Chyba: Daily quota exceeded

API Error: Daily quota exceeded (50,000 tokens)
Co dělat:

Počkej do zítřka. Pokud potřebuješ více — upgraduj na Google Cloud paid plan.

⚠️
Pro hobby projekt: FREE stačí

60 requesty za minutu = 3,600 za hodinu. To je hodně. Jen pokud intenzivně pracuješ — nemusíš se bát.

// sekce 04

Rate limiting — timeout čekání

Když je agent pomalý nebo čeká na API.

Chyba: Request timeout

Timeout: Request took longer than 30 seconds
Co se stalo:

Agent dělá moc velký úkol. Gemini API trvá dlouho odpovědět.

Řešení:

Rozděl úkol na menší kusy. "Přidej filtr" je OK, ale "Redesignuj celou aplikaci" je moc.

Chyba: Connection timeout (offline)

Error: Network timeout. Agent offline?
Řešení:

Zkontroluj internet. Zkus znovu. Pokud je stále offline — restartuj Antigravity.

Jak se vyhnout timeoutům

  • Menší úkoly — "Přidej button" ne "Udělej vše"
  • Bez velkých souborů v kontextu — pokud máš 10MB soubor, agent to nebude čít
  • Patience — někdy jde to pomaleji. Je to OK.
// sekce 05

Network & Connection chyby

Chyba: No internet

Error: Unable to reach Gemini API. Check your connection.
Řešení:

WiFi OK? Zkus: ping google.com v terminálu. Pokud není internet — restart routeru.

Chyba: 403 Forbidden (API key invalid)

Error 403: Invalid API key or permission denied
Co se stalo:

Tvůj Google účet není propojený nebo API key skončil.

Řešení:

File → Settings → Account → Reauth. Znovu se přihlas Google účtem.

Chyba: 500 Server error

Error 500: Gemini API server error
Řešení:

Není tvoje chyba. Google servery mají problém. Počkej 5 minut a zkus znovu.

// sekce 06

Jak diagnostikovat problém

Step 1: Co vidíš?

  • Chyba v Agent panelu? → Error message
  • Build padá? → Terminal output
  • Agent se vůbec neodpovídá? → Network issue

Step 2: Kde je chyba?

Terminal

Ctrl+` (backtick) → otevři terminal. npm run build a podívej se kde přesně selha.

Agent panel (vpravo)

Vidíš error message. Klikni "Details" pro víc info.

Browser console (DevTools)

F12 → Console → podívej se na JS chyby. Pokud je to runtime error, bude tam.

Step 3: Jak to diagnostikovat

ℹ️
Checklist diagnostiky

1. Přečti si error message — co říká
2. Google error →StackOverflow řešení
3. Řekni agentovi přesně co vidíš
4. Agent zkoušet opravit
5. Pokud nejde — revert (git checkout .)

// sekce 07

Checklist — co dělat když selže

Rychlý triage

  1. Přečti chybu — error message je tvůj průvodce
  2. Je to TypeScript? → Fixuj typ. Agent to často neví.
  3. Je to runtime? → Null check. if (data)
  4. Je to network? → Zkontroluj internet. Reauth.
  5. Je to rate limit? → Počkej. Nebo downsize úkol.
  6. Nic z toho? → Undo. git checkout .

Emergency Undo List

  • Necommitnuté změny: git checkout .
  • Commitnuté změny: git reset --hard HEAD~1
  • Srátit víc commitů zpět: git reset --hard HEAD~5
  • Specifický commit: git reset --hard abc123

Preventivní opatření

  • ✅ Commit PŘED tím než dáš agentovi úkol
  • ✅ Push na GitHub (backup)
  • ✅ .antigravity-rules soubor (limits agent)
  • ✅ Malé úkoly (timeout se vyhneš)
  • ✅ Čti approval plán (než schválíš)
✓
Není to strašné

Antigravity je bezpečné. Git ti vždycky umožní vrátit se. Zkouš, neboj se selhání.
