// sekce 01

Nový projekt vs Import — rozdíl

Nový projekt

Když nemáš nic — Antigravity ti vytvoří kostru projektu z šablony.

  • React template
  • Next.js template
  • Vanilla JS template
  • Prázdná složka

Import existujícího projektu

Máš projekt kde na disku (GitHub, lokálně, odkud chceš) — vezmeš ho a otevřeš v Antigravity.

  • Tvůj receptář (React project na disku)
  • Projekt z GitHubu (git clone + otevřít)
  • Projekt od kolegy (soubor ZIP či složka)
ℹ️
Pro tebe: Import je správná cesta

Máš receptář hotový na disku. Necháš ho tam, jen ho otevřeš v Antigravity. Nic se netkne.

// sekce 02

Vytvoření nového projektu

Pokud bys chtěl nový projekt od nuly:

1
Welcome tab — New Project

Když se Antigravity spustí, vidíš Welcome. Klikni "[New Project]"

2
Vyber template

React, Next.js, Svelte, prázdná — co chceš

3
Pojmenuj projekt

Třeba "my-app", "experiment-2024"

4
Vyber cestu

Kde na disku má být složka projektu (výchozí: ~/Projects)

5
Antigravity inicializuje

Vytvoří soubory, nainstaluje npm deps, spustí git init. Hotovo.

// sekce 03

Import existujícího projektu — TvůJ receptář

To je to co potřebuješ.

1
File → Open Folder

Nebo jednoduše drag-drop složku projektu do Antigravity

2
Vyber složku receptáře

Kde máš src/, package.json, .git atd.

3
Antigravity detekuje project

Vidí že je to React projekt, rozpozná .git, nastaví vše

4
Projekt je otevřený

File Explorer vlevo, editor uprostřed, agent vpravo. Hotovo!

Co se stane s tvým projektem?

  • ✅ Nic se nezmění — tvůj kód zůstane stejný
  • ✅ Antigravity jen čte soubory
  • ✅ Když uzavřeš Antigravity — projekt je stále tam
  • ✅ Můžeš ho otevřít v normálním VS Code kdykoliv
✓
Bezpečné — není to destruktivní

Import je jen otevření. Tvůj projekt si stojí sám o sobě.

// sekce 04

Workspace — více projektů zároveň

Antigravity umožňuje mít otevřeno více projektů najednou (na rozdíl od VS Code, kde máš jeden workspace).

Jak přidat druhý projekt do Workspace?

File → Add Folder to Workspace

Nyní máš v File Exploreru dva projekty vidět. Můžeš:

  • Otevřít soubor z projektu A
  • Editor práce — kód z A i B vidět vedle sebe
  • Agent vidí oba projekty v kontextu
⚠️
Workspace konfiguraci

Antigravity si pamatuje které projekty byly otevřené. Když zavřeš a znova otevřeš — jsou tam. Konfigurace je v .code-workspace souboru (podobně jako VS Code).

Pro tvůj případ

Máš jeden projekt (receptář) — workspace není nutný. Ale pokud bys chtěl přidat druhý experiment — takhle bys to dělal.

// sekce 05

Projekt Settings — per projekt

Každý projekt má svá settings — oddělená od globálních.

Kde jsou settings projektu?

File → Preferences → Settings, ale klikni na "Workspace" tab (ne "User")

Co se dá nastavit per projekt?

  • Agent Approval Mode — pro tenhle projekt Review-Driven, pro jiný Agent-Assisted
  • Ignored files — jaké soubory agent vidět nesmí (.env, temp/ atd.)
  • Build command — co se spustí příkazem npm run build
  • Format on Save — jen pro tenhle projekt
  • Extensions — které ext se mají v tomhle projektu aktivovat

Příklad: Tvůj receptář

"[receptar-workspace]": {
 "agent.approvalMode": "review-driven",
 "editor.formatOnSave": true,
 "files.exclude": {
  "node_modules": true,
  ".env.local": true
 }
}
ℹ️
Pro začátečníka: Defaultní settings jsou fajn

Nemusíš nic měnit. Antigravity si pamatuje čo potřebuje.

// sekce 06

Jak se vrátit z projektu

Zavřít projekt (ale ne Antigravity)

File → Close Folder

Projekt se zavře, vrátíš se do Welcome screenu. Ostatní otevřené projekty zůstanou.

Otevřít jiný projekt

File → Open Recent

Antigravity si pamatuje posledních 20 projektů které jsi otevíral. Můžeš se na ně vrátit jedním klikem.

Úplně zavřít Antigravity

File → Exit (nebo Ctrl+Q)

Vše se zavře. Ale projekty a Git data zůstanou na disku — nic se neztratí.

✓
Tvůj receptář bude existovat bez ohledu na Antigravity

Je to jen IDE — tvůj projekt je na disku. Antigravity je jen způsob jak s ním pracovat.

// sekce 07

Pro tvůj receptář — konkrétně

Tady je přesný postup:

Krok 1: Najdi složku receptáře

C:\Users\marti\Documents\rodinny-receptar
(nebo `/home/marti/projects/rodinny-receptar`)

Měla by obsahovat:
├─ src/
├─ public/
├─ package.json
├─ .git/
└─ vite.config.ts

Krok 2: Otevři v Antigravity

File → Open Folder → vyber složku receptáře

Krok 3: Ověř že je vše připravené

  • File Explorer vlevo vidí strukturu src/, public/
  • Terminal je dostupný (Ctrl+`)
  • Agent panel je vidět vpravo

Krok 4: Spusť dev server (pokud chceš)

Terminal: npm run dev

Server se spustí na http://localhost:3000
Agent ho vidí a můžeš s ním testovat

Krok 5: Pracuj s agentem

Agent → "Přidej filtr na cenu" → ... → hotovo

✓
Právě teď máš vše co potřebuješ

Otevřít receptář, pracovat s agentem, commitovat změny. Jednoduchý flow.
