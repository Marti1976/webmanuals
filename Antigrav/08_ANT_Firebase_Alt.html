Co potřebuješ — Frontend vs Backend

Tvůj receptář — co je co

  • Frontend — React komponenty, UI (to co vidí uživatel v prohlížeči)
  • Backend — API, databáze (pokud používáš)

Tvůj receptář konkrétně

Frontend: React PWA (receptář, filtry, UI) — Vercel nebo Netlify

Databáze (pokud chceš): JSON soubory nebo Supabase — Supabase

Bez backendu jde?

Ano! Pokud soubory ukládáš v prohlížeči (localStorage) nebo v JSON, nepotřebuješ backend.

ℹ️
Pro tvůj hobby projekt

Frontend hosting (Vercel) + případně Supabase na data. To je vše co potřebuješ.

// sekce 02

Frontend hosting — Vercel a Netlify

Ty dva jsou nejjednoduší a nejoblíbenější pro React projekty.

Vercel (doporučuji)

Vercel
Cena:FREE (s limity)
Build:Automatický z GitHubu
Domény:Gratis .vercel.app
Pro React:Ideální (made by Next.js crew)

Netlify (také dobrá)

Netlify
Cena:FREE (s limity)
Build:Automatický z GitHubu
Domény:Gratis .netlify.app
Flexibility:Vyšší (serverless functions)

Jak to funguje

  1. Přihlásíš se přes GitHub
  2. Vyberaš repository (tvůj receptář)
  3. Vercel/Netlify si vytáhne kód
  4. Spustí build (npm run build)
  5. Nasadí hotový projekt na jejich servery
  6. URL: rodinny-receptar.vercel.app
✓
Automatické deploye

Když pushneš do GitHubu — Vercel sám builduje a nasazuje. Není třeba nic dělat.

// sekce 03

Databáze — Supabase vs Firebase

Firebase končí. Potřebuješ alternativu pro data.

Supabase (TOP doporučení)

Supabase
Typ:PostgreSQL (relační DB)
Cena:FREE (bez expirů!)
Auth:Zabudovaná
API:REST + Real-time
Limits:Hojné na free

Firebase (porovnání)

Google Firebase
Typ:NoSQL (dokumenty)
Cena:FREE (Spark plan skončil!)
Auth:Zabudovaná (Google centric)
API:SDK (proprietární)
Limits:Velké na placený

Proč Supabase?

  • ✅ Free tier nemá expiraci
  • ✅ Open-source (PostgreSQL)
  • ✅ Snadné na použití (firebox-like API)
  • ✅ Real-time features (subscriptions)
  • ✅ Better docs a komunita

Jak začít se Supabase

  1. supabase.com → Sign up s GitHub
  2. Create project
  3. Dataase schema (vytvoř tabulky pro recepty)
  4. Get API key
  5. V React kódu: const { data } = await supabase.from('receptar').select()

// sekce 05

Bez GitHubu — jak to dělat

Pokud nechceš GitHub (nebo máš projekt jinde):

Možnost 1: ZIP → Vercel CLI

  1. Instaluj Vercel CLI: npm i -g vercel
  2. V složce projektu: vercel
  3. Vercel ti udělá build a nasadí bez GitHubu

Možnost 2: Docker → Railway

  1. Vytvoř Dockerfile v projektu
  2. Railway CLI: railway deploy
  3. Railway vytáhne z disku a nasadí

Možnost 3: Git bez GitHubu (Gitea, Codeberg)

Pokud chceš git ale ne GitHub:

  • Gitea — self-hosted GitHub
  • Codeberg — nonprofit GitHub alternativa
ℹ️
GitHub je nejjednoduší

Ale pokud ho nechceš — CLI tools fungují taky.

// sekce 07

Pro tvůj receptář — konkrétní setup

Fáze 1: Frontend na Vercel

  1. GitHub: push receptáře
  2. vercel.com → Sign up GitHub
  3. Import project → Select repository
  4. Vercel si vezme npm run build
  5. Hotovo! URL: rodinny-receptar.vercel.app

Fáze 2 (pokud chceš data): Supabase

  1. supabase.com → Sign up GitHub
  2. Create project → PostgreSQL database
  3. V SQL editortu: vytvoř tabulku receptar s columns: id, nazev, ingredience, cena
  4. Settings → API → Copy API key
  5. V React: npm install @supabase/supabase-js
  6. Použij Supabase client na čtení/zápis

Hotový setup

Frontend (React) → Vercel: rodinny-receptar.vercel.app

Backend (data) → Supabase PostgreSQL

Vše automaticky zálohované, věčně free.

✓
Tvůj receptář je v bezpečí

Na Vercel je to live, na GitHubu je backup, data jsou v Supabase. Perfekní kombinace.
